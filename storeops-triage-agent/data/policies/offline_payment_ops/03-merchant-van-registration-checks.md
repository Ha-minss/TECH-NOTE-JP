# [운영표준] 가맹점 및 외부 결제망 등록 상태 점검 지침

> 주의: 이 문서는 특정 회사의 실제 내부 절차가 아니라, 오프라인 결제 운영 문제를 다루는 AI Agent 데모를 위해 작성된 합성 운영 가이드입니다.

## 1. 목적

본 문서는 가맹점 번호 발급, 외부 결제망 또는 VAN 등록 상태, 등록 대기 상태 등으로 인해 발생할 수 있는 승인 오류를 점검하기 위한 절차를 안내합니다.

## 2. 적용 상황

- 가맹점 번호 미등록, 등록 대기, 등록 불가와 유사한 오류 문구가 확인된 경우
- 신규 오픈 매장에서 결제망 연동 또는 카드 승인 불가 문의가 접수된 경우
- 단말기 설치는 완료되었으나 결제 검증이 성공하지 않은 경우

## 3. 확인해야 할 항목

운영 담당자는 다음 항목을 확인하고 서로 대조해야 합니다.

- 내부 운영 시스템의 가맹점 심사 또는 등록 상태
- 외부 결제망 또는 VAN 등록 시스템의 등록 상태와 응답 이력
- 해당 매장의 유효한 가맹점 번호 존재 여부
- 등록 상태가 활성, 대기, 비활성, 미등록 중 어디에 해당하는지
- 최근 승인 실패 내역에 등록 관련 오류 문구 또는 응답 코드가 있는지
- 단말기 식별자와 결제 식별 설정의 정합성 여부
- 기기 설치 완료 이후 최초 결제망 테스트 또는 검증 성공 이력

## 4. 판단 시 주의사항

- 등록 관련 오류 문구가 있더라도 단말기 식별자와 결제 식별 설정 오류가 함께 존재할 수 있으므로 복합 요인을 고려합니다.
- 가맹점 등록 상태가 대기 또는 비활성인 경우, 최신 상태인지 확인하기 전에는 해결 완료로 판단하지 않습니다.
- 외부 결제망의 상태 조회가 지연되거나 오래된 정보일 수 있으므로 데이터의 최신성을 확인합니다.
- 등록 상태와 승인 실패 시각이 서로 맞지 않으면 단일 원인으로 확정하지 않습니다.

## 5. 사람 검토가 필요한 경우

다음 조건에 해당하는 경우 담당자 검토가 필요합니다.

- 외부 결제망 또는 VAN 등록 시스템의 상태 확인이 필요하나 내부 운영 데이터만으로 확인할 수 없는 경우
- 등록 상태가 대기 또는 비활성으로 보이지만 최신 상태 여부가 불확실한 경우
- 단말기 정보는 정상으로 보이나 등록 관련 승인 실패가 계속 발생하는 경우
- 가맹점 번호, 등록 상태, 승인 실패 내역 사이에 충돌이 있는 경우

## 6. 하면 안 되는 조치

- 등록 상태가 대기 또는 비활성임에도 2선 확인 없이 해결 완료 또는 정상 처리로 안내하지 않습니다.
- 외부 결제망 또는 VAN 담당자에게 사람 승인 없이 임의 이관하지 않습니다.
- 등록 관련 오류 문구만 보고 단말기 또는 결제망 중 하나의 원인으로 단정하지 않습니다.
