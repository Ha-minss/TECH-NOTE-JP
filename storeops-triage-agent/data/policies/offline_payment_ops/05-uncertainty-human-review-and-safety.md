# [운영표준] 불확실성 상황의 판단 유보 및 담당자 검토 지침

> 주의: 이 문서는 특정 회사의 실제 내부 절차가 아니라, 오프라인 결제 운영 문제를 다루는 AI Agent 데모를 위해 작성된 합성 운영 가이드입니다.

## 1. 목적

본 문서는 수집된 데이터의 부족, 로그 간 충돌, 조회 시스템 지연 또는 실패 등 불확실성이 높은 상황에서 운영 담당자가 준수해야 할 안전 수칙과 커뮤니케이션 기준을 정의합니다.

## 2. 적용 상황

- 필수 정보가 부족하여 원인을 명확히 규명하기 어려운 경우
- 가맹점 현장 설명과 운영 시스템 기록이 서로 충돌하는 경우
- 데이터 조회가 실패했거나 지연되어 최신 상태를 확정할 수 없는 경우
- 여러 원인 후보가 가능하지만 단일 원인으로 확정할 근거가 부족한 경우

## 3. 필수 확인 항목 및 운영 규칙

- 원인 규명을 위한 필수 정보가 누락된 경우 임의로 추정하지 말고 추가 정보를 요청합니다.
- 운영 시스템에서 확인 가능한 정량적 데이터와 가맹점주만 인지하고 있는 현장 상황을 분리하여 기록합니다.
- 시스템 로그와 현장 증거가 충돌할 경우 단일 원인으로 확정하지 않습니다.
- 필수 데이터 조회가 실패한 경우 확보된 제한적 데이터 안에서만 판단 결과를 제공합니다.
- 최신화되지 않은 지연 데이터나 과거 이력을 현재 사실처럼 안내하지 않습니다.
- 설정 변경, 결제 실행, 승인 취소, 환불, 외부 이관 등 민감한 조치는 사람 승인 없이 수행하지 않습니다.

## 4. 판단 및 안내 시 주의사항

명확한 판단이 어려운 경우 다음과 같은 표현을 사용합니다.

- "현재 확인된 근거만으로는 정확한 원인을 확정하기 어렵습니다."
- "정확한 파악을 위해 추가 확인이 필요한 항목은 다음과 같습니다."
- "일부 필수 데이터 조회가 실패하여 수동 확인이 필요합니다."
- "현장 설명과 운영 시스템 기록의 시간축이 충돌하므로 담당자 검토가 필요합니다."

## 5. 사람 검토가 필요한 경우

다음 조건에 해당하는 경우 담당자 검토가 필요합니다.

- 필수 정보가 부족하여 표준 가이드에 따른 해결 안내가 불가능한 경우
- 중요한 기록의 시간축이 서로 맞지 않는 경우
- 필수 데이터 조회 실패로 원인 후보를 확인할 수 없는 경우
- 설정 변경, 외부 결제망 확인, 설치 담당자 확인 등 사람 승인 또는 외부 조율이 필요한 경우

## 6. 하면 안 되는 조치

- 운영자 단독 판단으로 실 결제 실행, 승인 취소, 환불 처리, 중요 설정 변경을 수행하거나 유도하지 않습니다.
- 책임자 승인 없이 민감한 사안을 외부 제휴사, VAN사, 설치 담당자에게 임의 이관하지 않습니다.
- 근거가 부족한 상태에서 확정적 표현으로 원인을 안내하지 않습니다.
- 조회 실패를 정상 상태로 해석하거나, 오래된 데이터를 최신 사실처럼 안내하지 않습니다.
