# [운영표준] 신규 단말기 설치 및 식별 정보 검증 지침

> 주의: 이 문서는 특정 회사의 실제 내부 절차가 아니라, 오프라인 결제 운영 문제를 다루는 AI Agent 데모를 위해 작성된 합성 운영 가이드입니다.

## 1. 목적

본 문서는 신규 단말기 설치, 기존 단말기 교체, 단말기 활성화 직후 발생하는 결제 실패 또는 단말기 식별 관련 문의를 점검하기 위한 절차를 정의합니다.

## 2. 적용 상황

- 가맹점에 단말기를 신규 설치했거나 기존 단말기를 교체한 직후 결제가 되지 않는다는 문의가 접수된 경우
- 신규 단말기와 기존 단말기 중 어느 장비에서 문제가 발생하는지 구분이 필요한 경우
- 단말기 등록 정보와 현장 장비 정보가 일치하는지 확인해야 하는 경우

## 3. 확인해야 할 항목

운영 담당자는 다음 항목을 순서대로 확인하고 서로 대조해야 합니다.

- 운영 시스템에 등록된 해당 매장의 전체 단말기 목록
- 현장에 있는 기존 단말기와 신규 단말기의 논리적·물리적 구분
- 단말기별 결제 식별 설정값
- 단말기 번호, 시리얼 번호, 기기 고유 번호 등 물리 식별자
- 등록 시스템에 저장된 단말기 식별자와 현장 장비 식별자의 일치 여부
- 신규 단말기의 개시 또는 활성화 처리 시각
- 기존 단말기에서 발생한 마지막 정상 결제 시각과 최초 오류 발생 시각
- 최근 기기 교체, 설치 대행 방문, 설정 변경 이력

## 4. 판단 시 주의사항

- 결제 식별 설정 중복, 기기 식별자 불일치, 활성화 지연 문제는 서로 다른 가능성이므로 분리해서 검토합니다.
- 현재 시점의 설정값만 확인하지 말고, 문제가 발생한 시점의 설정값과 변경 이력을 함께 확인합니다.
- 신규 단말기 활성화 이후 기존 단말기 오류가 시작되었다면, 활성화 시점과 오류 발생 시점의 순서를 반드시 비교합니다.
- 특정 단말기에서만 오류가 발생하는지, 매장 전체 결제 경로에서 오류가 발생하는지 구분합니다.

## 5. 사람 검토가 필요한 경우

다음 조건에 해당하는 경우 담당자 검토가 필요합니다.

- 현장 장비의 물리 식별자와 등록 시스템의 식별자가 일치하지 않는 경우
- 교체 또는 설치 이력이 누락되어 어떤 단말기가 실제로 사용 중인지 확정할 수 없는 경우
- 문제 발생 시점의 설정 이력이 현재 설정과 달라 시간축 검토가 필요한 경우
- 단말기 설정 변경이 필요할 수 있으나 운영자 승인 없이 수행할 수 없는 경우

## 6. 하면 안 되는 조치

- 책임자 승인 없이 결제 식별 설정이나 단말기 식별자 정보를 수동 변경하지 않습니다.
- 단말기 등록 정보가 불확실한 상태에서 가맹점주에게 정상 처리 또는 해결 완료로 안내하지 않습니다.
- 기존 단말기와 신규 단말기의 구분이 불명확한 상태에서 특정 장비의 문제로 단정하지 않습니다.

## 7. 예시 문의

- "어제 새 단말기 설치했는데 기존 단말기에서 결제가 안 돼요."
- "기기를 바꾼 뒤부터 승인 오류가 떠요."
- "새 단말기는 되는 것 같은데 예전 단말기가 안 돼요."
- "단말기 번호가 맞는지 확인해달라고 안내받았어요."
