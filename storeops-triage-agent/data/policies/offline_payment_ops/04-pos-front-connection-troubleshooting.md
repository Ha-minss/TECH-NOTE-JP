# [운영표준] POS-결제단말기 연동 및 통신 장애 점검 지침

> 주의: 이 문서는 특정 회사의 실제 내부 절차가 아니라, 오프라인 결제 운영 문제를 다루는 AI Agent 데모를 위해 작성된 합성 운영 가이드입니다.

## 1. 목적

본 문서는 매장 내 POS 시스템에서 발생한 결제 요청이 결제 단말기 또는 Front 시스템으로 정상 전달되지 않는 로컬 통신 및 연동 문제를 점검하기 위한 절차를 다룹니다.

## 2. 적용 상황

- POS 화면에서 결제를 시도했으나 단말기로 결제 금액이 전송되지 않는 경우
- POS 결제 요청이 타임아웃되거나 전달 실패가 반복되는 경우
- 단말기는 켜져 있고 기본 상태가 정상으로 보이나 POS 연동 결제만 실패하는 경우

## 3. 확인해야 할 항목

운영 담당자는 다음 항목을 확인하고 서로 대조해야 합니다.

- POS와 결제 단말기 또는 Front 시스템 사이의 논리적 연결 상태
- POS와 단말기의 페어링 또는 연동 상태
- POS와 단말기가 동일한 로컬 네트워크를 사용하는지 여부
- 매장 내 IP 설정, 공유기, 방화벽 등 네트워크 환경
- 결제 요청 전송, 전달, 실패, 타임아웃 이력
- 단말기 전원 상태와 기본 네트워크 등록 상태
- 보안 키 다운로드 또는 Working Key 복구 필요 여부

## 4. 판단 시 주의사항

- 단말기 전원이 켜져 있고 정상 상태로 보이더라도 네트워크 이슈로 POS 요청 전달이 실패할 수 있습니다.
- POS 로컬 연동 실패 문제는 일반적인 카드 승인 거절 문제와 구분해서 분석합니다.
- 보안 키 누락, 네트워크 문제, 페어링 문제 중 하나로 섣불리 단정하지 않습니다.
- 연결 로그 또는 요청 전달 실패 기록이 없으면 POS-단말기 연동 문제로 확정하지 않습니다.
- 단말기 등록 상태와 외부 결제망 등록 상태가 정상인지도 필요한 경우 함께 확인합니다.

## 5. 사람 검토가 필요한 경우

다음 조건에 해당하는 경우 담당자 검토가 필요합니다.

- 연동 설정과 네트워크 상태가 정상으로 보이는데도 통신 타임아웃이 계속 발생하는 경우
- 요청 전달 실패가 의심되지만 관련 로그가 누락되어 확인이 어려운 경우
- 보안 키 다운로드 또는 Working Key 복구가 필요할 수 있으나 자동 처리하기 어려운 경우
- POS, 단말기, 네트워크 담당 영역이 분리되어 추가 조율이 필요한 경우

## 6. 하면 안 되는 조치

- 원인이 확인되지 않은 상태에서 단말기 공장 초기화나 전체 네트워크 재설정을 안내하지 않습니다.
- 연결 로그 없이 특정 키 문제나 특정 장비 문제로 단정하지 않습니다.
- 사람 승인 없이 결제 설정, 네트워크 설정, 보안 키 관련 조치를 수행하거나 유도하지 않습니다.

## 7. 예시 문의

- "POS에서 결제 눌러도 단말기로 금액이 안 넘어가요."
- "단말기는 켜져 있는데 POS 결제가 안 돼요."
- "결제 요청이 계속 타임아웃돼요."
- "POS랑 단말기가 연결이 끊긴 것 같아요."
