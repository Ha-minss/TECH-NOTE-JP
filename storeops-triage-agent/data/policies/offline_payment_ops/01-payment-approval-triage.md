# [운영표준] 오프라인 결제 승인 오류 초기 대응 지침

> 주의: 이 문서는 특정 회사의 실제 내부 절차가 아니라, 오프라인 결제 운영 문제를 다루는 AI Agent 데모를 위해 작성된 합성 운영 가이드입니다.

## 1. 목적

본 문서는 오프라인 가맹점에서 결제 승인 실패 및 오류 문의가 인입되었을 때, 1선 운영 담당자가 상황을 빠르게 분류하고 초기 대응하기 위해 확인해야 할 항목과 주의사항을 정의합니다.

## 2. 적용 상황

- 가맹점 현장에서 카드 결제 실패, 승인 거절, 단말기 화면 내 오류 문구 발생 등으로 결제 불가 문의가 접수된 경우
- 특정 단말기, 특정 결제 경로, 특정 결제 수단에서만 승인 실패가 반복되는 경우
- 최근 단말기 설치, 기기 교체, 설정 변경 이후 승인 오류가 발생했다는 문의가 접수된 경우

## 3. 확인해야 할 항목

운영 담당자는 문의 응대 시 다음의 정보를 우선적으로 수집하고 서로 대조해야 합니다.

- 오류가 발생한 정확한 시각 또는 발생 시간대
- 영향을 받은 결제 수단과 특정 카드사 또는 결제 수단에 국한되는지 여부
- 오류가 발생한 단말기 또는 결제 경로
- 오류 발생 시각을 기준으로 한 최근 승인 실패 내역
- 승인 응답 코드와 단말기 화면에 표시된 오류 문구
- 동일 매장 내 다른 단말기 또는 다른 결제 경로에서도 실패가 발생하는지 여부
- 최근 단말기 설치, 기기 교체, 가맹점 설정 변경 여부

## 4. 판단 시 주의사항

- 단일 오류 메시지만을 근거로 장애 원인을 확정하지 않습니다.
- 승인 실패 내역은 단말기 상태, 등록 정보, 최근 설정 변경 이력과 함께 대조해야 합니다.
- 가맹점주에게 반복 결제 시도를 안내하기 전, 장애 범위가 단일 매장·단일 단말기에 국한되는지 먼저 확인합니다.
- 오류가 특정 단말기 또는 특정 시점 이후에 집중되는 경우, 해당 단말기와 최근 변경 이력을 우선적으로 확인합니다.

## 5. 사람 검토가 필요한 경우

다음 조건에 해당하는 경우 2선 운영자 또는 담당 부서의 검토가 필요합니다.

- 오류 발생 범위가 단일 매장을 넘어 지역, 결제망, 네트워크 단위 문제로 의심되는 경우
- 확인 가능한 운영 데이터만으로 오류가 발생한 단말기나 결제 경로를 특정할 수 없는 경우
- 조치 후에도 동일한 원인 불명 오류가 반복되는 경우
- 승인 실패 로그와 현장 설명이 서로 충돌하는 경우

## 6. 하면 안 되는 조치

- 명확한 근거 없이 결제 재시도만 반복하도록 안내하지 않습니다.
- 원인이 확인되지 않은 상태에서 설정값 변경, 결제 취소, 환불, 재승인 등 민감한 조치를 수행하거나 유도하지 않습니다.
- 단일 로그나 단일 문구만으로 가맹점주에게 해결 완료 또는 정상 처리로 안내하지 않습니다.

- "단말기에 승인 오류라고 떠요."
- "아까부터 특정 카드만 결제가 안 되는 것 같아요."
- "새 장비를 설치한 뒤부터 결제가 잘 안 돼요."
